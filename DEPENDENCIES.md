### `zipdepth.onnx` — re-exported with the trained attention pooling

The default model was regenerated by [`tools/zipdepth_gcb_export.py`](tools/zipdepth_gcb_export.py)
because upstream's `scripts/export.py` substitutes the trained
`GlobalContextBlock` forward at export time. Upstream replaces GCNet-style
learned softmax attention pooling with a plain `avg_pool2d`, which also leaves
the trained `context_weight` convolution unused. Measured on the same
checkpoint over five natural photos, the substitution shifts the output by
**24.1 % of dynamic range** (43.5 % worst case); after the scale-and-shift fit
this application's relative-depth rendering applies, that is
**d1 0.998, Pearson 0.990, RMSE 2.74 % of range**.

The substitution is avoidable. The attention pooling's batched matrix multiply
is a weighted spatial sum, so `ReduceSum(x * softmax_mask)` is algebraically
identical while using only `Conv`/`Softmax`/`Mul`/`ReduceSum` — no `MatMul`,
which matters because a quantized `MatMul` is rejected on Hexagon v68
(`incorrect Value 68, expected >= 73`). The rewrite reproduces the trained
module to 1.4e-06.

Keeping the trained pooling is also the better choice *after* quantization,
which was not the expected result:

| A16W8 quantization of the same graph | d1 | Pearson |
|---|---|---|
| trained attention pooling (shipped here) | 0.945 | 0.952 |
| upstream `avg_pool2d` substitution | 0.887 | 0.921 |

Two caveats on this artifact:

- It is exported from `zipdepth_base_npu.pth`. Upstream's prebuilt
  `checkpoints/zipdepth_base_384x384.onnx`, which the previous binary derived
  from, does **not** reproduce from either released checkpoint — all four
  combinations of checkpoint and pooling deviate by more than one dynamic
  range, while parameter counts agree to within 27. This binary is therefore a
  **different model**, not a patched build of the previous one.
- The quantization figures above are CPU-simulated QDQ comparisons against the
  float graph. This binary has **not** been executed on SM8350 hardware.

Upstream's two other export-time substitutions (`StripPoolingAttention` and
`MinimalCrossScale`) were checked and are faithful — they only restate the same
arithmetic with static shapes for ONNX — and are preserved unchanged.

## Model variants

Runtime-selectable SM8350/v68 EPContext variants produced by the pipeline in
[`tools/variants/`](tools/variants/) (BatchNorm folding + QNN-specific QDQ
quantization, then QAIRT 2.48 `qairt-converter --preserve_io_datatype` and
`qnn-context-binary-generator --htp_socs sm8350`). A16/A8 is the activation bit
width; weights are signed INT8 per-channel throughout. The A8 variants exist for
speed measurement only — see the README warning about their output quality.
